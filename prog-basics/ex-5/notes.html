Exercise 5: Managing Environment Variables

Level 1: Basic Environment Variable Setup

Objective: Install dotenv and configure environment variables from a .env file (e.g., PORT, DB_HOST).

Steps:

  • Install dotenv package.
  • Create a .env file with configuration variables.
  • Load variables in the program using dotenv.

Outcome: The program reads and uses values from the .env file.

Level 2: Handling Missing Environment Variables

Objective: Add fallback values for missing environment variables and log warnings for missing variables.

Steps:

  • Add fallback values for variables in case they are missing.
  • Log a warning for missing variables (without exposing sensitive info).

Outcome: Missing variables are handled gracefully with fallback values, and warnings are logged when variables are missing.

Level 3: Secure Configuration with Environment Variables

Objective: Use environment variables to securely configure a database connection (SQLite) and API credentials, and avoid logging sensitive information.

Steps:

  • Use dotenv to configure SQLite and API credentials securely.
  • Ensure sensitive information is never logged directly.
  • Handle missing credentials securely, with appropriate warnings.

Outcome: Sensitive configuration details are handled securely, and missing credentials are flagged without exposure.
