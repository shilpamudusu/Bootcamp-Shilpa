Exercise Notes

Level 1: Basic Debugging with debug Module

Objective: Set up the debug module for logging messages in a single module.

Steps:

  • Install debug and set up a namespace.
  • Log messages using debug for debugging.
  • Use the DEBUG environment variable to enable or disable debug messages.

Outcome: Debug messages appear when DEBUG is set. You can control which modules' debug messages show by adjusting the environment variable.

Level 2: Multiple Modules with Separate Debug Namespaces

Objective: Use multiple modules, each with its own debug namespace, and control which modules log messages via environment variables.

Steps:

  • Create multiple modules, each with a unique debug namespace.
  • In the main program, import these modules.
  • Use the DEBUG environment variable to selectively enable debugging for specific modules.

Outcome: Debug messages are displayed for specific modules depending on the DEBUG setting.

Level 3: Combining debug with winston for Structured Logging

Objective: Integrate debug for debugging and winston for structured logging, and make the program interactive.

Steps:

  • Set up winston for logging with a structured format (like JSON).
  • Use debug for runtime debugging and winston for logging results.
  • Make the program interactive (e.g., accepting user input) and perform computations based on input.
  • Enable debugging using the DEBUG environment variable.

Outcome: Debug and log messages work together to provide both detailed debugging and structured logs.
