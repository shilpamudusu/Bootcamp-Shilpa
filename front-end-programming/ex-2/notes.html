Notes for Exercise 2: Using Tools

Objective:

  • Learn to package and structure projects using TypeScript.
  • Follow a development lifecycle: Development → Testing → Production.

Key Steps:

1. Setting Up a TypeScript Project:

  • Initialized a project using npm.
  • Configured tsconfig.json to support:
    • Modern module resolution.
    • JSON imports.

Challenges Encountered:

1. Importing JSON Files

Issue: Red underline in TypeScript for JSON imports.

Solution: Enabled the appropriate TypeScript option in tsconfig.json.

2. Table Rendering with Dynamic Data

Issue: TypeScript flagged type mismatches.

Solution: Defined an interface for strict typing.

3. Vite Build Error

Issue: "Could not resolve entry module 'index.html'."

Solution:

  • Moved index.html to the root directory.
  • Updated file paths for compatibility with Vite.

Outputs:

1. Functional Features:

  • Dynamically loads JSON data.
  • Renders an HTML table with headers and rows.
  • Sorting functionality added to headers.
  • Filtering rows via a text input.
  • Row highlighting on hover.

2. Development Lifecycle Tools:

  • Development: Live reloading with Vite.
  • Testing:
    • Linting with TypeScript.
    • Previewing the production build.
  • Production: Bundled output to the dist folder.

Final Output:

  • A fully functional, responsive table rendered dynamically using TypeScript.
  • Features include sorting, filtering, and row highlighting.
  • Production-ready build generated using Vite.

Reflection:

This exercise provided hands-on experience with TypeScript, project structuring, and modern development workflows. Despite challenges, the final output meets all objectives and prepares a strong foundation for handling similar tasks in real-world projects.
